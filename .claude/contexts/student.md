# Student Session Context

Paste this at the start of every AI chat session to give the AI the right context about who you are and what you need.

---

## How to Use This File

1. Copy the template below
2. Fill in the blanks (marked with `[ ]`)
3. Paste it as your first message to the AI
4. The AI will adjust its teaching style and responses accordingly

---

## Session Starter Template

```
I am a beginner web development student in the novice-dev-with-ai classroom.

About me:
- Current module: Module [X] — [Module Name]
- Experience level: [complete beginner / I know a little HTML / comfortable with HTML]
- Today's goal: [describe what you are trying to build or learn]

My current code (paste it here):
[paste your HTML and/or CSS]

What I've already tried:
[list what you attempted before asking]

My specific question:
[ask your focused question here]

How I want you to help:
- Explain the concept before showing code
- Give me hints before giving the full answer
- Tell me what I did well AND what needs improving
- Keep examples under 20 lines
- Comment every new concept in the code
```

## Rules for AI Sessions (Your Commitment)

By using this classroom's AI tools, you agree to:

**I will:**
- ✅ Type code myself — never copy-paste AI output without reading it
- ✅ Try to solve problems for at least 5 minutes before asking AI
- ✅ Read error messages carefully before asking what they mean
- ✅ Open Chrome DevTools and inspect my code visually before asking
- ✅ Ask specific questions, not vague ones
- ✅ Ask "why" when I don't understand something in the AI's response
- ✅ Commit my code to GitHub after every session

**I will not:**
- ❌ Ask the AI to write my whole project
- ❌ Ask the AI to "fix all my bugs" without trying first
- ❌ Copy-paste AI code I don't understand
- ❌ Use AI output to avoid learning the concept

## Good Prompt Templates (Save These)

### To learn a new concept:
```
Explain what [CSS Flexbox] is in plain English.
Use an everyday analogy before showing any code.
Then show me the smallest working example possible.
```

### To get unstuck on a bug:
```
My [element/layout/style] is [describe the problem].
I expected [X] to happen but [Y] happened instead.
I've already tried: [list your attempts].
My code is: [paste HTML] [paste CSS]
Give me a hint to point me in the right direction.
```

### To get a code review:
```
Please review my HTML/CSS using the beginner checklist.
Code: [paste]
Tell me: one thing I did well, and the single most important thing to fix.
```

### To check accessibility:
```
Check my page for these accessibility issues:
1. Alt text on all images
2. Labels on all form inputs
3. Heading order (one h1, no skipped levels)
4. Keyboard navigation
5. Color contrast
Code: [paste]
```

### When frustrated:
```
I've been working on [X] for [time] and I'm stuck.
I've tried: [list attempts].
I'm not asking for the answer — I just need you to explain
why [Y] doesn't work the way I expected.
```

## The 3-Before-AI Rule

Before opening the AI chat:

1. **Re-read the lesson notes** for this module in `curriculum/module-0X/lesson-notes.md`
2. **Try one more approach** on your own — what else could you try?
3. **Inspect in DevTools** — open F12, look at the Elements and Console tabs

Then ask with full context: what you tried, what you expected, what actually happened.

## Progression Checkpoint

Before starting each session, review your current milestone:

| Module | To Advance You Must... |
|---|---|
| 1 | Type the boilerplate from memory and explain each line |
| 2 | Build a 4-region page with zero `<div>` elements |
| 3 | Style a full page from scratch without referencing examples |
| 4 | Center any element + build a card row with Flexbox only |
| 5 | Build a 3-column grid that collapses to 1 column on mobile |
| 6 | Make a complete layout responsive: mobile, tablet, desktop |
| 7 | Identify and fix 5 accessibility issues in your own code |
| 8 | Build a form validator + dark mode toggle with vanilla JS |
